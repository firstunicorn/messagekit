fix(messaging): use explicit topic exchange in RabbitMQ publisher

The `RabbitEventPublisher` defaulted to using a string for the exchange
name (`"events"`). In FastStream, if an exchange name is passed as a string
to `RabbitBroker.publish`, it defaults to a direct exchange (or the default
typeless nameless exchange depending on configuration). 

When tests (and external consumers) declared the `"events"` exchange as a 
`topic` exchange, publishing a message with routing keys like `"user.created"`
might fail to route or get dropped if FastStream implicitly treated it as 
a direct exchange, causing `QueueEmpty` errors in tests.

Fixed by initializing `RabbitEventPublisher` with an explicit `RabbitExchange`
object configured with `ExchangeType.TOPIC` and `durable=True`.
